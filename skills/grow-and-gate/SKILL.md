# Grow & Gate

One job: stop the growth loop at the points where a human has to decide, and make
the decision well. The agent drafts, generates variants, and schedules. This skill
runs the gates.

The full checklist for every gate is in `reference/grow-gates.md` — read it when you
reach a gate. This skill is how you *operate* it. Unlike the build side, the grow
gates run in three cadences, so the first thing to establish is which one you're in.

## Always start with Gate ∞

Before anything, decide how much this piece earns. Do not run every gate on every
post.

1. Read the piece and size its **reach and permanence**. Ask the human: how far does
   this go, how long does it live, and how hard is it to walk back?
2. Pick the treatment:
   - **A reply, a quick post, something ephemeral and low-reach** → the minimum: a
     glance at voice (Gate 4) and send.
   - **A landing page, a launch, anything permanent or high-reach** → the full
     per-piece set.
3. State the chosen treatment before proceeding. That statement is the Gate ∞
   decision.

## Then: which cadence are you in?

- **Setting strategy** (new campaign, new audience, a fresh quarter) → work the
  **strategy** gates: G1 (audience + switching forces) and G6-define (what would "it
  worked" mean). These get set once and reused.
- **Making a specific artifact** → work the **per-piece** gates in order:
  G0 (ready to show?) → G2 (the one claim) → G3 (design) → G4 (copy) → G5 (channel,
  format, time).
- **Reading results** → work the **feedback** gate: G6-measure, and write the signal
  back for the build side (below).

Most sessions are per-piece. Confirm the strategy gates are already set for the
current campaign before making pieces against them — a per-piece run inherits G1 and
G6-define, so if they're missing, set them first.

## Working a gate

For each gate in the chosen set:

1. Name the gate and its lens (Design / Content / Positioning / Experience) from
   `reference/grow-gates.md`.
2. Present **the decision** the human is on the hook for — not a recommendation, the
   actual call.
3. Surface what the pipeline produced or claimed (the draft, the design, the numbers)
   and apply the **good judgment** heuristics for that gate.
4. **If the gate's lens isn't the human's**, run the "not your lens" fallback: put the
   agent to work making the decision defensible (map the switching forces, rank the
   claims, critique its own design against the taste profile, name what would prove a
   claim), and let the human decide against the structure. Do this *with* the human so
   they learn the gate, never instead of them.
5. Wait for the human's decision before moving on.

## The Foundation profiles (required)

This kit assumes the Foundation kit (second-brain-agent) is present. Read
`vault/Profiles/` before working the design and copy gates:

- **Gate 3 (design)** — check generated design against the **taste profile**. It's the
  standard that turns "does it look nice" into "is it ours."
- **Gate 4 (copy)** — draft, then rewrite against the **voice profile**, then
  fact-check every specific. Nothing publishes in a voice or a claim the profiles
  haven't been taught.

Without the profiles, Gates 3 and 4 collapse into guessing. If they're missing, tell
the human to run the Foundation kit's `profile-interview` first.

## The flywheel seam (Gate 6-measure)

At Gate 6-measure, after reading results against the Gate 6-define outcome, **write
the customer signal to the Foundation vault** (a note, or the capture/inbox path) so
the build side's Gate 0 reads it. The two kits form the loop through the vault: grow
learns what the market wants, the build side decides what to make next. Don't let a
real signal die in a dashboard — the write-back is what closes the loop.

## Rules

- **Never make a gate decision for the human.** Surface the call, apply the
  heuristics, show the evidence — then stop. Rubber-stamping polished slop or
  fabricated copy is the failure this skill exists to prevent.
- **Gate ∞ and cadence come first.** Most waste is running per-piece ceremony on a
  throwaway, or making pieces with no strategy set behind them.
- **Truth before polish, always (Gate 4).** A fabricated number or an unsourced claim
  ends the trust the whole channel runs on. If you can't source it, cut it.
- **Match rigor to reach, not to habit.** Gate ∞ resets it each time.
